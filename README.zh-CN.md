# 二叉树遍历可视化

轻量级静态网页：按指定节点数 **随机生成二叉树**，并一步步 **演示五种遍历**。侧栏、画布、底部播放条、中英切换等交互与 [AVLTreeVisualizer](https://github.com/layiku/AVLTreeVisualizer) 一致。

**[English README](README.md)**

## 功能

- **随机树**：节点数 \(N\) 取 1–31，点击 **生成树**。通过随机划分子树大小建树，节点值为打乱后的 `1..N`。
- **遍历**（任选一种后点 **开始演示**）：
  1. **前序** — 根 → 左 → 右  
  2. **中序** — 左 → 根 → 右  
  3. **后序** — 左 → 右 → 根  
  4. **层序（队列）** — 用队列做广度优先（非递归）  
  5. **层序（递归）** — 与 BFS 相同访问顺序，用深度与「按层」递归实现  
- **播放**：上一步 / 下一步、播放/暂停（自动步进）、速度滑块。  
- **可视化**：画布绘制，当前访问节点高亮。  
- **i18n**：切换中英文无需刷新页面。

## 算法说明（含示例）

下面用 **同一棵** 小树上的访问顺序，便于对比各种遍历。

**示例树**（数字为节点值；连线表示左 / 右孩子）：

```text
        1
       / \
      2   3
     / \
    4   5
```

- **结点 1** 为根；**2** 是 1 的左孩子；**3** 是 1 的右孩子；**4** 是 2 的左孩子；**5** 是 2 的右孩子。

### 二叉树

**二叉树**是每个结点 **至多有两个** 孩子（通常称为 **左**、**右**）的根树；空树没有结点。本应用通过随机划分子树大小生成 **随机** 二叉树，恰好 \(N\) 个结点；树的形状与哪一侧为空（若有）每次都可能不同。

*示例：* 上图是一棵 5 个结点的二叉树；结点 2 的孩子是 4、5，结点 1 的孩子是 2、3。

### 前序遍历

先访问 **当前结点**，再 **前序** 遍历左子树，再 **前序** 遍历右子树。常用于复制树或前缀式表示。

*示例树上的访问顺序：* **1, 2, 4, 5, 3**

### 中序遍历

先 **中序** 遍历左子树，再访问 **当前结点**，再 **中序** 遍历右子树。对 **二叉搜索树** 会得到有序序列（本演示生成的树不一定是 BST）。

*示例树上的访问顺序：* **4, 2, 5, 1, 3**

### 后序遍历

先 **后序** 遍历左子树，再 **后序** 遍历右子树，再访问 **当前结点**。常用于删除树或自底向上求表达式树。

*示例树上的访问顺序：* **4, 5, 2, 3, 1**

### 层序遍历（非递归 / 队列）

按 **层** 访问，自上而下、同一层从左到右。典型实现用 **队列**：队头出队并访问，若有左孩子则入队，再右孩子入队，重复直到队列为空。

*示例树上的访问顺序：* **1, 2, 3, 4, 5**

### 层序遍历（递归）

与上面 **访问顺序相同**，但用递归实现（例如按 **深度** 分层，对每一层扫描该层存在的结点）。本应用「层序（递归）」与队列版 **步序一致**，仅实现方式不同。

*示例树上的访问顺序（与非递归相同）：* **1, 2, 3, 4, 5**

## 如何运行

1. 克隆或复制本仓库。  
2. 用现代浏览器打开 `index.html`（Chrome、Firefox、Edge、Safari 等）。  
3. **仅使用**应用时无需构建或安装依赖。

## 自动化测试（Playwright）

端到端测试在 [`tests/`](tests/)：本地起静态服务（`serve`），在 Chromium 中打开页面，检查 UI 与页面内 `BinaryTreeLib` 行为。

**首次安装**（建议 Node.js 18+）：

```bash
npm install
npx playwright install chromium
```

**运行测试：**

```bash
npm test
```

其他命令：`npm run test:ui`（交互界面）、`npm run test:headed`（有头浏览器）、`npm run test:report`（打开 HTML 报告）。

向 `main` 或 `master` 推送 / 提 PR 时，仓库中的 [`.github/workflows/playwright.yml`](.github/workflows/playwright.yml) 会在 Ubuntu CI 上跑同一套测试。
